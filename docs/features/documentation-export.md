# Documentation export (`videodoc export`)

## Summary
`videodoc export <project> --format <format>` implements README §28, "Fase 15 — Export della documentazione". It packages generated Markdown sections from `docs/` into local export folders under `exports/`.

Supported formats:

- `markdown`
- `mkdocs`
- `docusaurus`
- `github-pages`
- `pdf`
- `html`

The exporter is local and dependency-free. It overwrites files it produces but does not delete export directories, so manually added files are not removed.

## Formats

`markdown` copies `outline.md`, generated section files and `review_report.md` when present.

`mkdocs` writes:

- `exports/mkdocs/mkdocs.yml`
- `exports/mkdocs/docs/index.md`
- one Markdown file per generated section

`docusaurus` writes a minimal docs scaffold:

- `exports/docusaurus/docusaurus.config.js`
- `exports/docusaurus/sidebars.js`
- `exports/docusaurus/docs/intro.md`
- one Markdown file per generated section

`html` and `github-pages` write static HTML pages generated by a small built-in Markdown renderer. `github-pages` also writes `.nojekyll`.

`pdf` writes `exports/pdf/documentation.pdf` using a small built-in PDF writer. It is intentionally simple: it preserves readable text, not rich typography.

## CLI

```bash
videodoc export corso-software-x --format mkdocs
# Project: corso-software-x
# +-------------------------------------+
# | Format | mkdocs                     |
# | Files  | 10                         |
# | Output | .../exports/mkdocs         |
# +-------------------------------------+
```

## Main Files
- `src/videodoc/core/services/export_service.py` — format-specific export packaging.
- `src/videodoc/cli/commands/export.py` — CLI command and `--format`.

## Tests
- Service: `tests/core/test_export_service.py`, covering all supported formats, missing generated sections and unsupported formats.
- CLI: `tests/cli/test_cli_export_command.py`.
