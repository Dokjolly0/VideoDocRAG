# Changelog

## Step 1 ŌĆö Project management foundation
- Added project initialization and a validated `config.yaml` schema ŌĆö see [features/project-initialization.md](features/project-initialization.md)
- Added the local project registry and portable project paths (`--path`, `list`/`link`/`unlink`/`path`) ŌĆö see [features/project-registry.md](features/project-registry.md)
- Fixed: `init` now registers projects under their canonical slug instead of the raw display name, matching `link`'s existing behavior ŌĆö see [features/slugify.md](features/slugify.md)
- Fixed: `init` on a path that already holds a *different* project now fails with a clear error instead of silently registering an unintended alias ŌĆö see [features/slugify.md](features/slugify.md)
- Hardened: a single malformed entry in the local registry can no longer crash `list`/`resolve`, and can always be removed via `unlink` ŌĆö see [features/project-registry.md](features/project-registry.md)

## Step 2 ŌĆö Source scanning and external paths
- Added support for external source paths: `paths.videos`/`attachments`/`codebase` can be an absolute path referencing material outside the project, with matching `init --videos`/`--attachments`/`--codebase` options ŌĆö see [features/external-source-paths.md](features/external-source-paths.md)
- Added `videodoc scan`, which enumerates a project's sources (respecting `config.scan` exclusions) and writes `sources.yaml` ŌĆö see [features/scan.md](features/scan.md)
- Fixed: `workdir`/`indexes`/`output`/`database` (and relative `videos`/`attachments`/`codebase`) now also reject `..` parent-traversal segments, not just anchored forms ŌĆö a relative value could otherwise still escape the project folder once joined with it ŌĆö see [features/external-source-paths.md](features/external-source-paths.md)
- Fixed: `ProjectService.init()` now validates a new project's `config.yaml` (including `--videos`/`--attachments`/`--codebase`) before creating any folder, so an invalid value never leaves partial project state on disk ŌĆö see [features/external-source-paths.md](features/external-source-paths.md)
- Fixed: added `core/utils/paths.py` as the single shared definition of path classification (`is_external_source_path`, `has_any_anchor`, `has_ambiguous_anchor`, `has_parent_traversal`), used by both config validation and path resolution/reporting (`resolve_source_path`, `SourceScanService._resolve_report`). Previously validation used `PureWindowsPath` while resolution used the platform-dependent `Path` ŌĆö harmless on Windows (where they agree) but the two could have silently disagreed on a non-Windows host ŌĆö see [features/external-source-paths.md](features/external-source-paths.md)
- Hardened: `scan`'s directory walker no longer silently drops subdirectories it can't read (e.g. permission denied) ŌĆö walk errors are now collected into `sources.yaml`'s new `scan_errors` field and surfaced as CLI warnings, never silently producing an incomplete-looking-complete manifest ŌĆö see [features/scan.md](features/scan.md)
- Fixed: `codebase_is_present()` no longer lets an unreadable-but-existing `codebase/` root (e.g. permission denied) crash `videodoc scan` with an uncaught exception before `scan_codebase()`'s own error handling could even run ŌĆö it now reports into the same `scan_errors` list ŌĆö see [features/scan.md](features/scan.md)
- Docs: the stale `sources.yaml` example in README ┬¦14 (which showed `config.yaml` content) was replaced with a real `SourceManifest`-shaped example, and the ┬¦15.1 `scan` output example now matches the actual CLI output (`Codebase: present (N files)`, `Excluded file patterns`)
